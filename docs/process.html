MindGraph 构建流程

把单作者文档构建成知识图谱 —— 评测驱动、循环迭代到收敛。LLM 步骤 确定性步骤（无需 LLM）

📄 输入：单作者文档（著作 / 讲稿 / 笔记，约 10 万–50 万字）
↓
1
结构化切分

过滤目录/页码/OCR 噪声 → 按标题分节 → ~900 字一段（不跨标题）→ 标注片段类型

确定性
↓
2
领域 schema 归纳

分窗扫全文，抽出该作者的核心概念 → 消歧合并 → 按频次排序，得到概念表 + 别名

LLM
↓
3
生成测试集

产出 ≥100 条分层 gold（抽取题 + 推理题），人工校验后冻结，作为后续度量基准

LLM + 人工
↓
⟲ 评测驱动的循环
4
两阶段抽取 → 写入 Neo4j

先逐片段抽节点 → 实体消歧 → 再在已知节点之间抽关系；写入图数据库

LLM（worker）
5
评测覆盖率

用冻结的测试集精确计算 节点覆盖率 / 关系覆盖率 / 回答得分

确定性
6
诊断缺漏

把差异归类：漏节点 / 漏关系 / 错连 / 粒度，并按类型统计，定位系统性问题

确定性
7
修正策略

据诊断给下一轮抽取注入针对性指令 + 类型级 few-shot（不含具体答案，防"教到测试集"）

规则
↺ 未收敛 → 回到步骤 ④ 再抽一轮　｜ ✓ 收敛：节点≥0.85 且 关系≥0.75 且 回答≥0.80，且不再提升
↓
🗂 输出：作者专属知识图谱 + 基于子图的问答 agent

编排大脑 = coding agent

造测试集、定/改 schema、跑流程、读报告决定是否再抽一轮。
无需 API key —— 它本身就是 agent（如 Claude Code / Codex）。

worker（推荐 DeepSeek V4 Pro）

实际干活：两阶段抽取 + 运行时回答。
唯一必需的 API key。

API judge（可选）

仅"自治模式"用于给回答打分。
默认 agent 驱动模式不需要。

固定不变：面向"观点/论证"的元本体（论断·论据·方法·条件·因果…）　｜　随文档变化：作者自己的概念与术语
